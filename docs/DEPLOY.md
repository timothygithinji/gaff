# Deploying Gaff from scratch

Gaff is a Cloudflare Worker (TanStack Start) backed by Neon Postgres, with background work
on Trigger.dev and infrastructure managed by Pulumi. Secrets live in Doppler and are
injected at runtime — nothing sensitive is committed. This guide takes you from a clone to a
running production instance.

It assumes the project was scaffolded with [`t-stack`](https://github.com/<org>/t-stack);
`t-stack provision` / `t-stack deploy` wrap the Pulumi + Wrangler steps. You can also run the
underlying commands directly (shown below).

---

## 1. Accounts you'll need

All have a usable free tier.

| Service | What it's for | Key(s) |
|--------|----------------|--------|
| [Cloudflare](https://cloudflare.com) | Workers runtime, R2, KV, DNS, Zero Trust Access | `CLOUDFLARE_API_TOKEN`, `CLOUDFLARE_ACCOUNT_ID` |
| [Neon](https://neon.tech) | Serverless Postgres | `DATABASE_URL` |
| [Trigger.dev](https://trigger.dev) | Scrape / cluster / enrich / email jobs | `TRIGGER_SECRET_KEY`, project ref |
| [Doppler](https://doppler.com) | Secret storage + runtime injection | a `gaff` project with `dev` + `prd` configs |
| [Pulumi](https://pulumi.com) | Cloudflare infra as code | `PULUMI_ACCESS_TOKEN` |
| [Anthropic](https://console.anthropic.com) | AI listing enrichment | `ANTHROPIC_API_KEY` |
| [Zyte](https://zyte.com) | Headless scraping (Zoopla) | `ZYTE_API_KEY` |
| [Google Maps Platform](https://console.cloud.google.com/google/maps-apis) | Geocode, Places, Routes | `GOOGLE_MAPS_API_KEY` (+ optional server key) |
| [EPC Open Data](https://epc.opendatacommunities.org) | Energy ratings | `EPC_OPENDATA_TOKEN` (`email:token`) |
| [Resend](https://resend.com) | Match + digest emails | `RESEND_API_KEY`, a verified sender |
| TfL / logo.dev | Optional: transit rate limits, brand logos | `TFL_APP_KEY`, `LOGODEV_TOKEN` |

You'll also want a domain you control in Cloudflare (for the custom hostname + Access gate),
though the default `*.workers.dev` URL works for a quick start.

## 2. Make it yours

A few identifiers are this-instance specific. Replace them before provisioning:

| File | Value | Set to |
|------|-------|--------|
| `t-stack.config.ts` | `org`, `domain` | your t-stack org slug + your hostname |
| `infra/cloudflare/Pulumi.production.yaml` | `accountId`, `zoneId`, `domain`, `resourcePrefix`, `accessEmailDomain`, `guestEmails` | your Cloudflare IDs, domain, a globally-unique R2 prefix, and who's allowed in |
| `wrangler.jsonc` | `routes[0].pattern` | your hostname (or delete the `routes` block for `*.workers.dev`) |
| `.github/workflows/deploy.yml` | `stack-name` | `<your-pulumi-org>/production` |
| Doppler `gaff/*` | `TRIGGER_PROJECT_REF` | your Trigger.dev project ref (`proj_…`) |
| Doppler `gaff/prd` | `EMAIL_FROM` | a Resend-verified sender, e.g. `Gaff <gaff@updates.your-domain.com>` |

The KV namespace ID and R2 bucket name in `wrangler.jsonc` are placeholders — Pulumi fills
them in on first provision. `.t-stack/state.json` is regenerated by provisioning.

## 3. Secrets

Put every key from **[`.env.example`](../.env.example)** into Doppler:

- `gaff/dev` — your local development values (a Neon dev branch, test keys).
- `gaff/prd` — production values. Also add `PULUMI_ACCESS_TOKEN` here (used by provisioning,
  stripped before the Worker secret sync).

The app validates the full set on first request (`src/lib/env.ts`), so a missing **required**
secret fails fast with a readable error. Optional secrets degrade gracefully.

For CI (`.github/workflows/deploy.yml`), add these as **GitHub repo secrets**:
`DOPPLER_TOKEN` (a `gaff/prd` service token), `CLOUDFLARE_API_TOKEN`, `CLOUDFLARE_ACCOUNT_ID`.
The Cloudflare token needs full provisioning scope (Workers, KV, R2, **Zero Trust Access**,
and **Account API Tokens**) because Pulumi mints an R2 token and manages the Access app.

## 4. Local development

```sh
bun install
bash scripts/setup-doppler.sh          # wires Doppler to this repo (gaff/dev)

bun run dev                            # web + Trigger.dev dev server
bun run db:migrate                     # migrate your Neon dev branch
bun run db:studio                      # browse the DB
```

`scripts/neon-env.ts` gives each git branch its own Neon branch, so migrations and test data
stay isolated. `bun run dev:prod` runs the local web server against the **prod** Doppler
config and database — handy for debugging, gated to localhost in dev only.

## 5. Provision infrastructure

```sh
t-stack provision
```

This runs the Pulumi program in `infra/cloudflare/`, which creates:

- a **KV namespace** (`KV` binding — Access JWKS cache),
- an **R2 bucket** (`BUCKET` binding — cached listing photos) plus a bucket-scoped S3 API
  token for the Trigger.dev workers, synced back into `gaff/prd` as the `R2_*` secrets,
- a **Cloudflare Access** application gating the whole domain to your `accessEmailDomain` /
  `guestEmails`, plus a public **bypass** on `/clusters/*` so listing photos render in email,
- the **custom-domain route** and an Image-Resizing-off zone setting.

It then patches `wrangler.jsonc` with the real binding IDs and records state in
`.t-stack/state.json`.

## 6. Deploy

```sh
bun run db:migrate:prod                # apply migrations to prod Postgres
bun run deploy                         # vite build + wrangler deploy
```

…or just push to `main` — `.github/workflows/deploy.yml` provisions infra, syncs Worker
secrets from Doppler, applies migrations, and deploys, in that order.

Deploy the Trigger.dev tasks too (CI does not): from the Trigger.dev project,
`bunx trigger.dev@latest deploy`. Re-run this whenever a task in `src/trigger/` changes.

## 7. Redeploying later

- **Code change** → push to `main` (or `bun run deploy`).
- **Schema change** → `bun run db:generate`, commit the migration; CI applies it before deploy.
- **New secret** → add to Doppler; the deploy's secret-sync step pushes it to the Worker.
- **Infra change** (`infra/cloudflare/`) → `t-stack provision` (or it runs in CI).
- **Trigger task change** → redeploy the Trigger.dev tasks (see above).

## Troubleshooting

- **Every page 500s after deploy** — usually a required secret missing from the Worker.
  Check `doppler secrets --project gaff --config prd --only-names` against `src/lib/env.ts`.
- **Pulumi 403 during provision** — the Cloudflare token is missing a scope (Account API
  Tokens, Zero Trust Access, or Zone Settings). Widen it and re-run.
- **Photos don't load in emails** — the `/clusters/*` Access bypass app didn't apply; check
  the Access dashboard for the `…-photos` application.
